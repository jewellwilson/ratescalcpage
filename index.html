Rate Method: Arithmetic Mean


Description


Arithmetic mean is a simple average of the rates all rate engines included in the calculation.The rates from each rate engine are summed and divided by the number of rate engines.

There is no weighting done by number of rate reports.


This option will calculate a rate for a given load by summing the total rates for each rate engine and dividing by the number of rate engines. This method does not consider the amount of data (rate reports or matching loads) for each rating engine, and the results therefore do not represent the average rate independent of the rate engine


Third-Party Rate Sources


Forecasting (FTR)

Forecasting data from FTR is presented separately in the application and cannot be added to a calculated rate setup. Therefore, it can not be be used in rate formulas.


Chainalytics

Both Chainalytics Contract Rates and Spot Rates engines can be included in a calculated rate setup. If Chainalytics is included, the rates are treated like any other engine except the number of rate reports will always be "1". Chainalytics only generates one rate, which is benchmarked based upon numerous industry cost drivers.


Note: Although a "1" will be displayed in calculations, a dash will be displayed in all on screen rate search results.


EXAMPLE:

Rate Engine Average Rate Rate Reports Weight (%)
My Average Rate $1,200.00 15 75
RTF Average Rate $1,000.00 150 5
Chainalytics Spot Rate 1 1 10
Chainalytics Contract Rate 1 1 10

Rate Method: Weighted Mean


Description


The weighted mean considers the number of rate reports when performing the average calculation so that rate engines with more reports are weighted more heavily in the calculation.

Chainalytics

Both Chainalytics Contract Rates and Spot Rates engines can be included in a calculated rate setup. If Chainalytics is included, the rates are treated like any other engine except the number of rate reports will always be "1". Chainalytics only generates one rate, which is benchmarked based upon numerous industry cost drivers.


Note: Although a "1" will be displayed in calculations, a dash will be displayed in all on screen rate search results.


EXAMPLE:

Rate Engine Average Rate Rate Reports Weight (%)
My Average Rate $1,200.00 15 75
RTF Average Rate $1,000.00 150 5
Chainalytics Spot Rate 1 1 10
Chainalytics Contract Rate 1 1 10

Formulas


$$ \text{Wighted Mean} = { \text{(Rate Engine 1 Rate x # of Reports) + (Rate Engine 2 Rate x # of Reports) + (Etc.)} \over \text{Rate Engine 1 Reports + Rate Engine 2 Reports + Etc.} }$$


Calculation


$$ { \text{( 1,200.00 x 15 ) + ( 1,000.00 x 150 ) + ( 1,000.00 x 1 ) + ( 900.00 x 1 )} \over \text{( 15 ) + ( 150 ) + ( 1 ) + ( 1 )} } = {$1,017.37}$$

Rate Method: Custom


Description


Custom adds a user-defined weighting to the average so that both rate reports and custom weighting are considered in the calculation. The custom option gives the user the ability to customize the importance of each rate engine to meet their needs.


Third-Party Rate Sources


Forecasting (FTR)

Chainalytics

If Chainalytics is included, the rates are treated like any other engine except the number of rate reports will always be "1". Chainalytics only generates one rate, which is benchmarked based upon numerous industry cost drivers.


Note: Although a "1" will be displayed in calculations, a dash will be displayed in all on screen rate search results.


EXAMPLE:

Rate Engine Average Rate Rate Reports Weight (%)
My Average Rate $1,200.00 15 75

Formulas


$$ \text{Custom Mean} = { \text{(Rate Engine 1 Rate x # of Reports x Weight) + (Rate Engine 2 Rate x # of Reports x Weight) + (Etc.)} \over \text{Rate Engine 1 Reports x Weight + Rate Engine 2 Reports x Weight + Etc.} }$$


Calculation


$$ { \text{(1,200.00 x 15 x 0.75 ) + (1,000.00 x 150 x 0.05 ) + (1,000.00 x 1 x 0.10 ) + (900.00 x 150 x 0.10 )} \over \text{( 15 x 0.75 ) + ( 150 x 0.05 ) + ( 1 x 0.10) + ( 1 x 0.10 )} } = {$1,118.25}$$
